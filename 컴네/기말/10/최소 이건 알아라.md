# 🚀 **데이터링크계층 필수 핵심 정리 (이것만 알면 전체가 잡힘)**

(디테일은 나중에 보고, 지금은 중요한 기둥들만!)

---

# 1. ✔ 링크 계층의 역할 (가장 중요)

링크 계층은 **같은 네트워크(LAN) 안에서 한 홉(one hop) 전송을 담당**한다.

### 핵심 4가지 기능

1. **Framing (프레이밍)**

* IP 패킷을 프레임 형태로 감싸서 전송하는 기능.

2. **MAC (Media Access Control)**

* 다수의 노드가 같은 매체를 공유할 때 충돌 없이 전송하기 위한 규칙.
  예) CSMA/CD, CSMA/CA, ALOHA

3. **오류검출 / 정정(Error Detection/Correction)**

* CRC, 패리티, 체크섬 등
* *Ethernet은 CRC로 "검출만" 하고 정정은 안 함.*

4. **신뢰적 전송(Reliable Delivery, 선택적)**

* 무선(Wi-Fi)은 링크 자체에서 재전송 제공
* 유선(Ethernet)은 보통 고속이므로 오류만 검출하고 재전송은 위 계층에 맡김

---

# 2. ✔ “한 홉(one-hop) 전송”의 의미

* 링크 계층은 **바로 연결된 이웃 노드 간** 데이터 전송만 담당함.
* PC → 스위치, 스위치 → 라우터, 라우터 → 라우터
* 여러 홉을 가는 end-to-end 전송은 전송/네트워크 계층 역할.

---

# 3. ✔ 오류 검출 기술(반드시 암기)

링크 계층의 핵심 기능 중 하나.

### 주요 방식

* **Parity bit**: 한 비트 오류 검출
* **Checksum**: UDP/tcp 계층에도 사용되는 단순 합계 기반 검출
* **CRC (Cyclic Redundancy Check)**
  → 가장 강력함. Ethernet, Wi-Fi 전부 CRC 사용
  → “프레임 끝에 붙는 FCS(Field Check Sequence)”

이렇게 3개 차이는 꼭 구분해야 한다.

---

# 4. ✔ MAC 주소 (48bit) – 링크 계층의 주소

* NIC 카드에 굽혀져 있는 **하드웨어 주소 (48비트)**
* 전 세계적으로 유일해야 함
* IP 주소와 역할이 다름

| 구분     | MAC 주소         | IP 주소        |
| ------ | -------------- | ------------ |
| 계층     | 링크 계층(L2)      | 네트워크 계층(L3)  |
| 주소 종류  | 물리 주소          | 논리 주소        |
| 누가 할당? | NIC 제조사가 부여    | DHCP/수동 설정   |
| 사용 범위  | 같은 LAN 안에서만 사용 | 전 세계 라우팅에 사용 |

MAC = 주민등록번호
IP = 우편주소
라고 비유할 수 있음.

---

# 5. ✔ ARP (Address Resolution Protocol) – L2와 L3 사이 연결

역할: **IP 주소 → MAC 주소로 변환**
(LAN 내부 통신을 위해 반드시 필요!)

### 동작 흐름

1. ARP Request (브로드캐스트)
   “192.168.1.20의 MAC 주소 누구야?”

2. 해당 호스트만 ARP Reply (유니캐스트)
   “내 MAC은 3C:5A:…”

3. ARP Cache에 저장 → 이후엔 브로드캐스트 안 함

이 내용은 매우 자주 시험에 나오는 핵심.

---

# 6. ✔ Ethernet 프레임 구조 (가장 중요한 표)

프레임은 다음처럼 구성됨:

```
[Dst MAC][Src MAC][Type][Payload(IP Datagram)][FCS(CRC)]
```

* **Type 필드**: 안에 어떤 상위 프로토콜이 있는지 표시
  예) 0x0800 = IPv4, 0x86DD = IPv6, 0x0806 = ARP
* **FCS**: CRC 기반 오류 검출 32bit

---

# 7. ✔ 스위치(Switch)의 동작 원리 (매우 중요)

스위치는 **MAC 주소 기반 포워딩 장비 (L2 장비)**

### 3가지 핵심 동작

1. **Learning(학습)**

   * 들어온 프레임의 출발지 MAC을 MAC 테이블에 저장
   * MAC → 어느 포트로 연결되는지 기록

2. **Forwarding(전달)**

   * 목적지 MAC이 테이블에 있으면 그 포트로만 전달

3. **Flooding(플러딩)**

   * 목적지 MAC을 모르면 모든 포트로 뿌림
   * 응답이 오면 학습하여 테이블 업데이트

스위치는 Plug & Play 가능 → 관리자가 MAC을 설정할 필요 없음.

---

# 8. ✔ VLAN (Virtual LAN)

물리적 스위치 하나를 여러 개의 논리적 LAN으로 나누는 기술.

### 주요 개념

* **Access Port**: 한 VLAN만 통신
* **Trunk Port**: 여러 VLAN을 태그(Tag)해서 전달

### VLAN Tagging (802.1Q)

프레임 안에 VLAN ID(12bit)를 추가해
“이 프레임은 VLAN 10 소속입니다” 라고 표시.

---

# 9. ✔ CSMA/CD vs CSMA/CA (시험 단골)

### Ethernet (유선): CSMA/CD

* 충돌 감지
* 충돌 발생 시 즉시 중단 + Exponential Backoff
* 하지만 현대 스위치는 full-duplex 환경 → 충돌 없음 → CSMA/CD 의미 감소

### Wi-Fi (무선): CSMA/CA

* 충돌을 미리 회피하는 방식
* 무선은 충돌 감지가 불가능하기 때문

---

# 10. ✔ NIC (Network Interface Card)

링크 + 물리 계층 기능을 수행하는 하드웨어.

### NIC가 하는 일

* 프레임 캡슐화/디캡슐화
* MAC 주소 관리
* CRC 계산
* 물리적 신호 변환

즉, CPU가 아니라 **NIC 자체가 링크 계층 대부분을 수행한다.**

---

# 🎯 최종 핵심 요약 (이것만 외우면 단원 전체가 잡힘)

1. 링크 계층 = **한 홉 전송, 프레임 단위**
2. 프레임 = MAC 주소 기반 전달
3. 오류 검출 = CRC, 체크섬, 패리티
4. MAC 주소 = 장치 고유 L2 주소 (48bit)
5. ARP = IP → MAC 변환 (브로드캐스트 → 유니캐스트)
6. 스위치 = MAC 학습 + 포워딩 + 플러딩
7. VLAN = 스위치를 논리적으로 분리 (802.1Q 태그)
8. CSMA/CD (Ethernet), CSMA/CA(Wi-Fi)
9. NIC = 링크 계층 핵심 기능 담당
